# Markdown
## 一、啥是Markdown
Markdown是一种轻量级标记语言，创始人是约翰·格鲁伯。它允许人们使用易读易写的纯文本格式编写文档，然后转换成有效的XHTML(或者HTML)文档。这种语言吸收了很多再电子邮件中已有的纯文本标记的特性。(如果你认真读完了，那么恭喜你，这段来自百度百科，你可以百度一下，再读一遍)。
  
按我的理解，就是使用一些特定的符号，诸如：# = - ...等符号来修饰你的文本内容，使其能够在一些平台上被转换成XHTML或HTML的格式，更好地做内容输出。
   
Markdown具体的产生和历史，感兴趣的小伙伴可以自行了解。详情可参见：  
[younghz/Markdown](https://github.com/younghz/Markdown.git)  
[菜鸟教程](https://www.runoob.com/markdown/md-tutorial.html)   

本文主要是记录Markdown的常见用法，以防作者“提前进入老年期”，忘东忘西，便于日后自己查找使用。

*PS：接下来在文中将使用**MD**来代指Markdown。*

## 三、内容

### 1.标题

 1级标题 使用 =  
 2级标题 使用 -   
 **使用特点**：需在文本中占据两行，第1行是标题内容，第2行是**至少3个 = 或 -** 
> **一级标题**
>> 一级标题   
>>`=======`
>
> **效果：** 
> 
>> 一级标题
>> ======

> **二级标题**
>> 二级标题   
>>`=======`
>
> **效果：** 
> 
>> 二级标题
>> ------

使用`#`表示1-6级标题。 
> **1-6级标题**
>> `#`  一级标题   
>> `##`  二级标题   
>> `###`  三级标题   
>> `####`  四级标题   
>> `#####`  五级标题   
>> `######`  六级标题   
>
> **效果：**
>> # 一级标题
>> ## 二级标题
>> ### 三级标题
>> #### 四级标题
>> ##### 五级标题
>> ###### 六级标题

### 2.段落                                                  
段落的格式就是直接写，没有什么特殊的符号标识。
注：在段落文本中，若需要换行的话要使用 **2个以上空格(space)+回车键(Enter)**
      
> **段落**
> 我是段落内容，巴拉巴拉巴拉...   
> 我是段落内容，巴拉巴拉巴拉...   
> 我是段落内容，巴拉巴拉巴拉...   
> **效果：**
> 
我是段落内容，巴拉巴拉巴拉...  
我是段落内容，巴拉巴拉巴拉...  
我是段落内容，巴拉巴拉巴拉...


***

### 3.文本强调
在需要被强调的文本前后添加`*`、`_`。

> **斜体**
>> `*斜体*，_斜体_`
>
> **效果：**
>> *斜体*,_斜体_

> **粗体**
>> `**粗体**，__粗体__`
>
> **效果：**
>> **粗体**，__粗体__

> **粗斜体**
>> `***粗斜体***，___粗斜体___`
>
> **效果：**
>> ***粗斜体***，___粗斜体___
***
### 4.分隔线

在一行中使用 **3个以上的`*`、`-`、`_`** 来建立一个分隔线。  
除了星号、减号和下划线之外，行内不能有其他内容，符号之间的空格会被忽略。

> `***` *`(星号 空格会被忽略，例如：* * * 也会被识别为分隔线)`*
***
> `---` *`(减号 空格会被忽略，例如：- - - 也会被识别为分隔线)`*
---
> `___` *`(下滑线 空格会被忽略，例如：_ _ _ 也会被识别为分隔线)`*


***

### 8 列表
MD支持`无序列表`和`有序列表`。
无序列表使用单个的星号(`*`)、加号(`+`)、减号(`-`)来表示。符号与列表内容需有空格隔开。

> **无序列表`* 星号`**
>> `* 列表内容 1`
>> `* 列表内容 2`
>> `* 列表内容 3`
>>
>**效果：**
>> * 列表内容
>> * 列表内容
>> * 列表内容

> **无序列表` + 加号`**
>> `+ 列表内容 1`
>> `+ 列表内容 2`
>> `+ 列表内容 3`
>>
>**效果：**
>> + 列表内容
>> + 列表内容
>> + 列表内容

> **无序列表` - 减号`**
>> `- 列表内容 1`
>> `- 列表内容 2`
>> `- 列表内容 3`
>> 
>**效果：**
>> - 列表内容
>> - 列表内容
>> - 列表内容

有序列表使用数字加`.`来表示，使用空格与列表内容隔开。

> **有序列表**
>> `1. 列表内容 1`
>> `2. 列表内容 2`
>> `3. 列表内容 3`
>> 
>**效果：**
>> 1. 列表内容 1
>> 2. 列表内容 2
>> 3. 列表内容 3

**列表嵌套**可以使我们的列表内容更具有层次划分和结构清晰的特点。
使用时，子列表选项前添加 **4个空格(space)**。

> **嵌套列表**
>> `1. 流程 1`
>> &nbsp;&nbsp;&nbsp;&nbsp;`- 流程 1 子列表`
>> &nbsp;&nbsp;&nbsp;&nbsp;`- 流程 1 子列表`
>>
>> `2. 列表内容 2`
>> &nbsp;&nbsp;&nbsp;&nbsp;`- 流程 2 子列表`
>> &nbsp;&nbsp;&nbsp;&nbsp;`- 流程 2 子列表`


>**效果：**
>> 1. 流程 1
>>     - 流程 1 子列表
>>     - 流程 1 子列表
>>
>> 2. 流程 2
>>     - 流程 2 子列表
>>     - 流程 2 子列表

### 9 区块引用
区块引用是段落的开头使用`>`+`空格`，再添加文本内容。

 **区块引用**

\> 文本内容  
\> 文本内容

 **效果：**
> 文本内容
> 文本内容
>
<br>

 **区块嵌套**

\> 区块-第1层  
\>> 区块-第2层  
\>>> 区块-第3层

  **效果：**
> 区块-第1层
>> 区块-第2层
>>> 区块-第2层

### 10.代码

使用**反引号 ( \` )** ，添加在代码的前后。

 **代码**
> \`System.out.println("Hello World")\`
 
 **效果**
 
 `System.out.println("Hello World")`

<br>

在代码文本前使用 **4个空格(space)** 或 **制表符(Tab)**

  **代码区块**
 > 正常文本位置
 >
 > &nbsp;&nbsp;&nbsp;&nbsp; public class HandsomeBoy {
 > &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;private final String name = "bill4h";
> &nbsp;&nbsp;&nbsp;&nbsp; }
 >
 >正常文本位置
 
  **效果**

    public class HandsomeBoy{
        private final String name = "bill4h";
    }

<br>

使用**3个反引号(\```)**包裹代码,可以指定一种语言（非必须）。

 **3个反引号\```**
> \```java
> &nbsp;&nbsp;&nbsp;&nbsp; public class HandsomeBoy {
> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;private final String name = "bill4h";
> &nbsp;&nbsp;&nbsp;&nbsp; }
> \```

**效果：**
```java
public class HandsomeBoy{
    private final String name = "bill4h";
}
```

### 11 链接
链接的使用方法：
> **方法一：**
> `[链接名称](链接地址)`
>
>
>> `我的github：[bill4h](https://github.com/bill4h)` 
>> 
> **效果**
>> 我的github：[bill4h](https://github.com/bill4h) 

> **方法二：**
>`<链接地址>`
> 
>
>> `<https://github.com/bill4h>`
>
> **效果**
>> <https://github.com/bill4h>

> **方法三：**
> `[链接名称][链接变量]`
>>
>> `我的github：[bill4h][my_github]`
>> *`......(文中其他内容)`*
>> `[my_github]:https://github.com/bill4h` *`(文档末尾)`*
>>
>> *(这里的链接变量的赋值要放在文档的末尾，且不会显示在文档中）*
>>
> **效果**
>> 我的github：[bill4h][my_github]
